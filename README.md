Artemis Financial is a client that needed improvements to the security of its software system, especially to protect sensitive financial data and reduce vulnerabilities in their application. The goal of the project was to identify security issues in the code and improve the system’s overall safety and reliability.

In this project, I did well in identifying key vulnerabilities such as missing authentication, weak handling of data, and insecure dependencies. I also learned how important secure coding is for protecting user information and maintaining trust in a system. Software security adds value by reducing risk, preventing attacks, and protecting both the company and its users.

The most helpful part of the assessment was learning how to systematically review code for different types of vulnerabilities. The most challenging part was understanding the real-world impact of each issue.

To improve security, I added layers such as better dependency management, improved data handling, and secure coding practices. In the future, I would continue using tools like OWASP guidelines and vulnerability databases to help evaluate and mitigate risks.

After refactoring, I tested the application to make sure it still worked correctly and reviewed the code again to ensure no new issues were introduced.

Some useful tools and practices included OWASP resources, Java security libraries like MessageDigest, and general testing and debugging in Eclipse. From this assignment, I could show employers my vulnerability report and secure version of the application as proof of my ability to identify and fix security issues.
